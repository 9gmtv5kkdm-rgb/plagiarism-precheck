# 投稿前自查清单与查重工作流

> 本文档提供系统化的查重预检工作流程——从初稿完成到正式投稿的全程管理。
> 所有示例使用虚构文本段落，不包含任何真实论文内容。
> 本文档与论文工坊的其他技能形成联动：paper-review-methodology（审查）、academic-writing-bank（重写模板）、reference-formatter（引用修正）。

---

## 一、投稿前自查清单（6 项）

在提交论文至期刊或查重系统前，完成以下 6 项检查：

### ✅ 第 1 项：查重报告完整阅读

- [ ] 已获取最新查重报告（CNKI / Turnitin / 其他指定系统）
- [ ] 已检查"总文字复制比"并确认在目标阈值以下
- [ ] 已检查"去除引用复制比"（关键指标）
- [ ] 已逐一查看所有标红区域的位置和来源
- [ ] 已检查"单篇最大复制比"不超过 5%
- [ ] 已保存完整查重报告 PDF 备查

### ✅ 第 2 项：AI 检测通过

- [ ] 已使用至少一个 AI 检测工具（Turnitin AI Detection / GPTZero / 其他）
- [ ] 全文 AI 生成概率在可接受范围（建议 < 20%）
- [ ] 已重点关注 AI 率 > 30% 的段落
- [ ] 所有"一段落同时标红 + AI 率高"的双高区域已完成强制性重写
- [ ] AI 使用声明已加入论文（如适用）

### ✅ 第 3 项：引用全面匹配

- [ ] 正文中每条引用在参考文献列表中都有对应条目
- [ ] 参考文献列表中每条文献都在正文中被引用
- [ ] 引用格式全文一致（APA / MLA / GB/T 7714 / 期刊指定格式）
- [ ] 直接引用均已加引号并标注页码
- [ ] 间接引用（paraphrase）均已标注出处
- [ ] 自我引用比例 ≤ 20%

### ✅ 第 4 项：数据来源可溯源

- [ ] 所有统计数据标注了来源和年份
- [ ] 所有图表标注了"数据来源"或"作者自制"
- [ ] 问卷调查数据已说明施测时间、对象和有效样本量
- [ ] 实验数据可追溯至原始记录
- [ ] 第三方数据（行业报告等）已获使用权（如适用）

### ✅ 第 5 项：改写记录可追溯

- [ ] 对高重复率段落的每次改写均有记录（改写前文本 → 改写后文本）
- [ ] 最终版本与任何单一来源不构成大段连续匹配
- [ ] 改写后文本经过至少一轮"冷却重读"（间隔 24 小时以上再自行审读）
- [ ] 改写记录包含改写依据（使用了策略金字塔的哪一级策略）

### ✅ 第 6 项：格式终版检查

- [ ] 论文格式符合目标期刊/学位论文模板要求
- [ ] 所有文件和文件名规范（不含特殊字符和空格）
- [ ] PDF 生成后检查字体嵌入和图表清晰度
- [ ] 文件名不包含作者信息（盲审适用时）
- [ ] 已创建最终提交版本的文件副本

## 二、查重后分类处理流程

查重报告到手后，不要慌。按以下分类系统处理每一处标红：

### 2.1 标红区域分类

| 类型 | 特征 | 典型位置 | 处理方式 |
|---|---|---|---|
| **A. 引用未标注** | 文字与他人文献匹配，但没有引用标注 | 文献综述段、背景介绍 | → 添加规范引用 |
| **B. 方法描述** | 实验方法、统计方法的标准化描述 | 方法论章节 | → 简化+个性化（见 2.2） |
| **C. 文献综述** | 对已有研究的概括性表述 | 文献综述章 | → 按主题重组（见 2.3） |
| **D. 数据分析** | 数据讨论和结果的解读 | 讨论/结论章 | → 优先核查是否误标（见 2.4） |
| **E. 套话/模板** | "本研究具有重要理论意义"等 | 结论章 | → 删除套话，写具体贡献（见 2.5） |

### 2.2 A 类处理：添加规范引用

**如果内容确实来自他人文献但你忘记标注：**

1. 确认来源文献
2. 若为间接引用（用自己的话概括），添加"（作者，年份）"
3. 若为直接引用，添加引号 + 页码
4. 更新参考文献列表

**但需注意：** 即使添加了标注，大段间接引用仍可能被判定为学术原创性不足。若该段落的引用密度过高（超过该段文字的 60% 都能追溯到单一来源），建议并行使用降重策略 1-3。

### 2.3 B 类处理：方法描述个性化

方法描述的标红难以完全避免——实验器材型号、统计方法名称等是无法改变的事实。但描述性文字可以优化：

**优化前（高度标准化，容易标红）：**
> "本研究使用 SPSS 26.0 对数据进行分析。首先进行描述性统计分析，计算各变量的均值、标准差。然后进行信度检验，采用 Cronbach's α 系数衡量问卷的内部一致性。最后进行回归分析，检验研究假设。"

**优化后（加入选择理由和细节）：**
> "数据分析通过 SPSS 26.0 完成。考虑到本研究的变量构成包含连续型指标和分类指标，描述性统计阶段分别计算了均值和标准差以及频次分布。问卷信度以 Cronbach's α 评估——预测试阶段各分量表 α 值均高于 0.78，满足了 Nunnally（1978）建议的 0.70 基准。研究假设的验证采用分层回归，以控制人口学变量的潜在混淆效应。"

**关键改进：**
- 加入"为什么要用这个方法"的理由
- 加入具体的阈值和基准
- 用预测试的细节替代通用描述

### 2.4 C 类处理：文献综述重组

文献综述段是查重标红的重灾区。根本原因不是抄袭，而是"综述写法高度趋同"。

**核心方法：按主题组织而非按作者罗列**（详见 `rewriting-strategies.md` 策略 3）。

### 2.5 D 类处理：数据分析段误标

数据分析段通常原创性最高，标红较可能是：
- 误标（与某篇文献偶然一致）
- 自引漏标（与自己的前期研究表述一致）

**处理步骤：**
1. 查看标红来源——如果是偶然匹配且无实质相似，可标注为可接受的巧合
2. 如果是自引漏标，添加"（本文作者此前已发表，见 X et al., 202X）"
3. 微小调整句式使连续匹配被打断

### 2.6 E 类处理：结论段套话清理

删除所有"本研究具有重要理论意义和实践价值"类套话，改为具体贡献陈述。

> 详见 `rewriting-strategies.md` 4.4 节。

## 三、多轮查重管理

### 3.1 文件命名规范

每一轮查重和降重都保存独立版本，确保可追溯：

```
论文标题/
├── 查重/
│   ├── v1_初稿_2025-03-01.docx
│   ├── v1_查重报告_2025-03-05.pdf
│   ├── v2_降重后_2025-03-10.docx            ← 基于 v1 降重
│   ├── v2_查重报告_2025-03-12.pdf
│   ├── v3_二次降重_2025-03-18.docx          ← 基于 v2 降重
│   ├── v3_查重报告_2025-03-20.pdf
│   └── vF_终版_2025-03-25.docx              ← 最终提交版
└── 降重记录/
    ├── v1→v2_降重对照表.docx                 ← 记录每次改写的原文→改文对照
    └── v2→v3_降重对照表.docx
```

### 3.2 多轮降重的节奏控制

| 轮次 | 目标 | 核心动作 |
|---|---|---|
| 首次查重 | 定位问题区域 | 完整查重 + AI 检测 + 阅读报告 |
| 第一轮降重 | 消除高重复率段落 | 分类处理 A-E 类标红，重点解决双高段落 |
| 二次验证 | 确认降重效果 | 查重 + AI 检测，对比首轮报告 |
| 第二轮降重 | 细调边界段落 | 处理重复率接近阈值的单句/短语 |
| 终版查重 | 获取官方结果 | 使用正式提交渠道查重，保存官方报告 |

### 3.3 降重节奏提醒

- **避免"一次性查重"心态**：首次查重后至少预留 2-3 轮迭代时间
- **冷却期原则**：每轮降重后至少间隔 24 小时再进行下一轮验证，让"新鲜感"消退后再审视自己的改写
- **不追零重复率**：绝对零重复率既不现实也不必要，5-8% 通常是健康的原创与引用平衡区
- **保存所有中间报告**：在终版查重出现意外时可以回溯定位问题

## 四、与论文工坊其他技能的联动

本技能（`plagiarism-precheck`）是论文工坊技能生态的一部分，与其他技能形成完整的工作流闭环：

### 联动 1：→ `paper-review-methodology` #23 脱落筛查

当论文审查流程执行到 #23 脱落筛查（AI 痕迹分布检查）时：
- 脱落筛查发现的高 AI 率段落 → 触发本技能的"双重检测流程"
- 确认"查重标红 + AI 率双高" → 标记为 P1 级问题
- P1 级问题 → 强制执行策略 1-3 级重写

```
paper-review-methodology #23
      │
      ├── 发现 AI 率异常段落
      │
      └──→ 触发 plagiarism-precheck 双重检测
              │
              ├── 双高 → P1 级，强制重写
              └── 仅 AI 高 → 去特征化处理
```

### 联动 2：→ `academic-writing-bank` 重写模板

当制定降重策略后需要具体执行时：
- 根据被标红段落的文体类型（定义段/综述段/方法段/讨论段）
- 从 writing-bank 中选取对应场景的模板句式
- 结合本技能的降重策略金字塔进行改写

### 联动 3：→ `reference-formatter` 引用修正

查重后发现的 **A 类问题（引用未标注）**：
- 补全引用标注后
- 使用 reference-formatter 统一检查和修正引用格式
- 确保全文中、英文引用的格式一致性

### 联动 4：→ `humanizer` / `humanizer-zh` 文本去 AI 化

双重检测后发现 **AI 率高但查重不标红**的段落：
- 使用 humanizer 技能进行文本去特征化
- 重点关注句式多样性、术语密度变化和学术"呼吸感"的恢复

## 五、快速参考卡

### 降重优先级排序

| 优先级 | 关注区域 | 行动 |
|---|---|---|
| 🔴 P0 | 查重标红 + AI 率双高 | 从零重写（策略 1-3） |
| 🟠 P1 | 单篇复制比 > 5% | 多源融合改写 |
| 🟡 P2 | 去除引用后仍 > 阈值 | 分类处理 A-E |
| 🟢 P3 | 局部短句标记 | 句式微调（策略 4-5） |
| ⚪ 不处理 | 专有名词、术语、数据 | 保留原样 |

### 降重效果自评检查

完成降重后，用以下问题自评：

1. 被改写的段落是否比原文**更清晰**？（降重的副产品应该是更好的表达）
2. 改写后的段落是否**准确**传达了原始观点的含义？
3. 读者能否分辨哪些是你的观点、哪些是引用他人的观点？
4. 这篇论文读起来像**一个作者**写的，还是多个作者拼凑的？
5. 如果把改写段和原文对比，是否构成 **"不同的表达"而非"不同的词"**？
